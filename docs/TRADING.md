# Торговий модуль (`app/trading`) — MVP автоматичної торгівлі

Автоматичний торговий бот для крипти. Архітектура абстрагована від конкретної
біржі через [ccxt](https://github.com/ccxt/ccxt) (100+ бірж єдиним API), тож її
легко масштабувати на інші біржі та ринки.

> ⚠️ **ДИСКЛЕЙМЕР.** Жоден торговий бот не гарантує прибуток. Алгоритмічна
> торгівля несе ризик **повної втрати капіталу**. Цей код — для особистого
> використання та навчання. Порядок безпечного запуску: **paper → testnet →
> малі реальні суми**. Не торгуй грошима, які не готова втратити.

## Стратегії (`TRADE_STRATEGY`)

| Значення | Тип | Логіка входу (LONG) |
|----------|-----|---------------------|
| `ema_rsi` (деф.) | тренд | бичий EMA-кросовер + RSI не перекуплений |
| `macd` | тренд | лінія MACD перетинає сигнальну вгору |
| `bollinger` | повернення до середнього | ціна нижче нижньої смуги, вихід на середній |
| `donchian` | пробій | пробій N-періодного максимуму |

Спільне для всіх:
- **Розмір позиції** рахується від ризику (`RISK_PER_TRADE`), а не «на скільки вистачить».
- **SL/TP** на основі ATR — адаптивні до волатильності.
- **Захист:** ліміт одночасних позицій + денний стоп-ліміт збитку.

> Різні стратегії підходять різним ринкам: трендові (`ema_rsi`, `macd`) — для
> ринку, що рухається; `bollinger` — для бічного; `donchian` — для сильних рухів.
> Який краще для конкретної пари — покаже бектест/оптимізатор.

## Spot vs ф'ючерси, плече та шорти

| `TRADE_MARKET_TYPE` | Що це | Можливості |
|---------------------|-------|------------|
| `spot` (деф.) | звичайна купівля активу | лише **long**, без плеча, без ліквідацій |
| `swap` | безстрокові ф'ючерси | **long + short**, плече (`TRADE_LEVERAGE`) |

- `TRADE_ALLOW_SHORTS=true` (тільки swap) — сигнал «sell» відкриває **шорт**,
  тож бот може заробляти й на падінні ринку.
- `TRADE_LEVERAGE` — плече для swap. ⚠️ **Збільшує і прибуток, і ризик
  ліквідації.** Починай з `1`, підвищуй свідомо й малими кроками.
- Ризик на угоду (`RISK_PER_TRADE`) лишається головним запобіжником: розмір
  позиції рахується так, щоб втрата на стопі не перевищила заданий відсоток —
  незалежно від плеча.

> Спот — безпечний старт (максимум втрати = вартість позиції). Ф'ючерси з плечем
> дають більше можливостей, але вимагають дисципліни в ризику.

## Підбір параметрів (оптимізатор)

```bash
# Перебирає сітку параметрів обраної стратегії на історії та ранжує
# за ризик-скоригованим показником (дохідність / просадка):
TRADE_STRATEGY=macd python -m app.trading.optimize BTC/USDT 1500
```
Виводить ТОП-комбінацій і друкує найкращу одразу у вигляді env-змінних —
лишається скопіювати їх у `.env`. ⚠️ Підбір на історії ≠ гарантія на майбутнє
(ризик перенавчання) — кращі параметри додатково перевіряй у paper-режимі.

## Швидкий старт

```bash
pip install -r requirements.txt

# 1) Перевір стратегію на історії (нічим не ризикуєш)
python -m app.trading.backtest BTC/USDT 1500

# 2) Запусти paper-режим (реальні ціни, віртуальні гроші)
export DATABASE_URL=postgres://...        # та сама БД, що й основний застосунок
export TRADE_MODE=paper
python -m app.trading.runner              # безкінечний цикл
python -m app.trading.runner once         # один прохід (для крону)

# 3) Статус через HTTP / веб-дашборд
curl localhost:8000/trading/status
#  відкрий у браузері:  http://localhost:8000/trading/dashboard
```

### Дашборд

`GET /trading/dashboard` — веб-сторінка з кривою капіталу, картками
(капітал, дохідність, PnL за сьогодні, win-стат) і таблицею останніх угод.
Автооновлення кожні 15 c. Дані беруться з `trade_equity` / `trade_positions`.
Перемикання режиму: `?mode=paper` або `?mode=live`. Графік малюється через
Chart.js з CDN (жодних додаткових залежностей).

## Перехід на реальні гроші (коли впевнена)

```bash

export TRADE_API_KEY=...                   # ключ БЕЗ права на вивід коштів
export TRADE_API_SECRET=...
# export TRADE_API_PASSWORD=...            # для okx/kucoin
```

Рекомендація: створюй API-ключ **тільки на торгівлю**, без права виводу,
і за можливості з IP-білим списком.

## Деплой на Render

У `Procfile` доданий процес `trader`. Запусти його як окремий
**Background Worker** (не web), з тими ж env-змінними та `DATABASE_URL`.
Альтернатива без постійного воркера — Render Cron, що періодично викликає
`python -m app.trading.runner once` або `POST /trading/run`.

## Куди масштабувати далі

- **Інші біржі/ринки** — змінити `TRADE_EXCHANGE` (ccxt вже підтримує); для
  форексу/акцій додати окремий адаптер `market.py` (MT5/Alpaca/OANDA).
- **Більше стратегій** — `strategy.py` зроблений як один клас; легко додати
  реєстр стратегій і запускати кілька паралельно на різних парах.
- **Шорти та ф'ючерси** — додати `side="short"` у брокер і відповідні ордери.
- **Краще керування ризиком** — трейлінг-стоп, кореляційні ліміти, волатильнісне
  масштабування позиції.
- **Веб-дашборд** — крива капіталу та історія угод уже є в БД (`trade_equity`,
  `trade_positions`), лишилось намалювати графік.
